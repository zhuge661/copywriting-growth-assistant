# 使用人配置 / Creator Profile

> 本文件决定 skill 为谁服务、写什么话题、用什么口吻、在哪些平台交付。
> 开始协作前必须先读取本文件。开源版本不内置任何个人 IP 的私有资料、样本、固定结尾或专属话题轨道。

---

## 当前默认配置

开源版本默认不预设具体使用人。第一次使用时，必须先填写下面的「可复制空白模板」。

如果没有填写以下字段，禁止直接开始创作：

- 使用人 / IP 名
- 账号名
- 核心受众
- 默认平台
- 默认话题轨道
- 固定口播结尾
- 固定品牌标签
- 英文固定口播结尾
- 英文品牌标签
- 英文目标受众 / 地区
- 强禁忌

## 使用前检查

- 如果没有明确当前使用人，先让使用人填写配置。
- 如果没有填写话题轨道，不要替使用人发明长期内容矩阵；先补齐。
- 如果没有填写固定结尾和品牌标签，交付配套时用占位提醒，不要替使用人编。
- 如果要做英文或双语内容，但没有填写英文固定结尾、英文品牌标签、英文目标受众，先提醒补齐。
- 如果使用人提供了专属规则文件，只读取当前使用人的专属规则，禁止读取其他 IP 的规则或样本作为默认依据。

## 迁移规则

可以迁移：

- 先查证、再消化、确认内核后落稿。
- 完整模式 / 半快出模式 / 快出模式。
- 备料输出四段：事实、可打角度、风险点、建议不展开。
- 中英双语解析机制。
- 样本库成长机制。
- 自检清单结构。

必须重填：

- 使用人身份、账号名、受众、平台。
- 话题轨道和各轨道 KPI。
- 固定结尾、品牌标签。
- 英文固定结尾、英文品牌标签、英文目标受众。
- 口吻尺度、禁忌、可蹭和不可蹭的话题。

默认不迁移：

- 任何个人 IP 的固定标语。
- 任何个人 IP 的话题轨道。
- 任何个人 IP 的样本库结论。
- 任何个人 IP 的专属表达规则。
